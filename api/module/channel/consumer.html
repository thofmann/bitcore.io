new Consumer(opts)

Params

  • opts Object
    • network string | Network - ‘livenet’ or ‘testnet’
    • expires number - unix timestamp in millis since epoch
    • [commitmentKey] bitcore.PrivateKey - key to use when negotiating the channel
    • [providerPublicKey] string - the public key for the server, in hexa compressed format
    • [providerAddress] string - the final address where the server will be paid
    • [fundingKey] bitcore.PrivateKey - key to use for funding the channel
    • [refundAddress] string - address to use for refund/change

consumer.network

Either ‘livenet’ or ‘testnet’

Type: bitcore.Network

consumer.expires

The expiration date for the channel, in seconds since UNIX epoch

Type: number

consumer.commitmentKey

This is the key used for the 2-of-2 locking of funds

Type: bitcore.PrivateKey

consumer.providerPublicKey

Type: bitcore.PublicKey

consumer.providerAddress

The address where the server will be paid.

Type: bitcore.Address | string

consumer.fundingKey

A private key for funding the channel. An alternative implementation could
provide a list of unspent outputs and the keys needed to sign the outputs

Type: bitcore.PrivateKey

consumer.fundingAddress

The address where the user will pay to fund the channel

Type: bitcore.Address

consumer.refundAddress

The address where the refund will go to (also used for the change)

Type: bitcore.Address

consumer.refundKey

If no refund address is provided, a private key is generated for the user

Type: bitcore.PrivateKey

consumer.commitmentTx

The commitment transaction for this channel

Type: Commitment

consumer.processFunding(utxo)

Adds an UTXO to the funding transaction. The funding transaction exists
merely because we can’t expect the wallet of the user to support payment
channels.

Params

  • utxo Object

consumer.setupRefund()

Build the refund transaction (TX 2)

Returns: bitcore.Transaction

consumer.validateRefund(messageFromProvider)

Validates that a message contains a valid signature from the Provider
that allows the Consumer to spend the lock transaction (TX 1)

Params

  • messageFromProvider string - JSON-serialized message

Returns: boolean - true if the signature is valid

consumer.incrementPaymentBy()

Increments the amount being paid by a given amount of satoshis.

Returns: bitcore.Transaction - the updated transaction

consumer.getPaymentTx()

Idiomatic shortcut to retrieve the payment transaction.

Returns: bitcore.Transaction
